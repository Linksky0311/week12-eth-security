============================================================
  VULNERABILITY EXPLANATION
============================================================
  Root cause: 'external interaction before state update'
    withdraw() sends ETH (external call) BEFORE zeroing balances[msg.sender].
    This violates the Checks-Effects-Interactions (CEI) pattern.

  Attack steps:
    1. Attacker deposits 1 ETH  → DAO total: 11 ETH
    2. Attacker calls withdraw() → DAO sends 1 ETH to attacker
    3. attacker.receive() fires  → balance[attacker] still = 1 ETH (not yet 0!)
    4. receive() calls withdraw() again → DAO sends another 1 ETH
    5. Steps 3-4 repeat until DAO is empty
    6. Stack unwinds: unchecked balance decrements execute (no revert)

  Fix: zero balances[msg.sender] BEFORE the external call (CEI).
